VM OVH Cloud

¿Qué pasa, amigo?

  • Les recettes sont souvent trop différentes de la prod
  • Complexe de répondre à certains besoins clients (ftp, ...)
  • Pouvoir gérer des outils spécifiques facilement
  • Eviter qu'un dev chez nous fasse tomber nos recettes.
VM OVH Cloud

Lo que necesita?

  • Des VM quand on veut, facilement, rapidement
  • Le plus iso prod possible : php, mysql, tools, config, etc
  • Rendre la chose automatisable à terme
VM OVH Cloud

Una solución, señor?

  • Oui, regarde le titre du slide.
  • Différents snapshots de serveurs préconfigurés
  • Des VM up & configurées en moins de 5 minutes
  • Une API REST pour automatiser les tasks : up, down, snapshot, reset, ...
VM OVH Cloud

¿Problemas, señor?

  • Une fois une VM configurée avec un snapshot, remettre à jour le snapshot de base est compliqué.
  • Une maintenance longue si on veut appliquer quelque chose partout

¡Pero!

  • Néanmoins utilisés depuis plusieurs mois sur des projets !
  • ... Ok, surtout de la TMA.
VM OVH Cloud

Y en 2017?

  • Séparer les datas des serveurs même : faciliter l'upgrade des serveurs
  • Automatiser la gestion : créer le subdomain, la VM, backup via snapshot, timer d'inactivité, ...
  • Intégrer à Robo ?

Coding Style Guide

PSR2

  • Uniforme sur tous les pôles
  • Assure une meilleure lisibilité du code

CSP

  • Ajoute une couche de sécurité côté navigateur
  • Très simple à mettre en place
  • Possibilité de l'utiliser comme simple logger
  • Dévélopement plus rigoureux 
header('Content-Security-Policy', 'directives');
  • Définit régles sécurité à appliquer, ex: pas de inline non signé
  • Simple composition de directive dans un header http
  • header définissant un end point de reception de log
  • Nous empêche d'écrire du js inline non signé (par un nonce ou autre)
